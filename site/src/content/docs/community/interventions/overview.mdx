An **intervention handler** is a composable control layer that intercepts agent lifecycle
events and returns typed decisions — proceed, deny, guide, confirm, or transform. Handlers
are registered via the `interventions` option in agent configuration and evaluated in order,
with short-circuiting on deny or guide actions. See
[Interventions](../../user-guide/concepts/agents/interventions/) for the full concept and
built-in handlers.

The SDK ships reference handlers like
[Cedar Authorization](../../user-guide/concepts/agents/interventions/cedar-authorization.md),
[Steering](../../user-guide/concepts/agents/interventions/steering.md), and
[Human-in-the-Loop](../../user-guide/concepts/agents/interventions/human-in-the-loop.md).
The packages below are **community-built** intervention handlers you can install and attach
to an agent.

:::note[Community maintained]
These packages are maintained by their authors, not the Strands team. Review packages
before using them in production. Quality and support may vary.
:::
